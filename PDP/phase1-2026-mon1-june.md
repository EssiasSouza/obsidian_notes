# June 2026 - Phase 1 Execution Plan

## Monthly Objective

The objective for June 2026 is to establish strong technical foundations for the entire long-term roadmap.

This month is intentionally focused on fundamentals because deep understanding of Linux, Networking and Kubernetes Core concepts will accelerate every future topic, including cloud architecture, security, platform engineering and distributed systems.

The focus during this month is not speed.
The focus is technical clarity and confidence.

The intention is to:

* reduce foundational insecurity
* improve troubleshooting capability
* strengthen infrastructure reasoning
* build hands-on operational confidence
* create sustainable study discipline
* improve technical English exposure

This month should create the baseline for the following phases.

This month follows the 70-20-10 model:

| Category      | Percentage | Purpose                                      |
| ------------- | ---------- | -------------------------------------------- |
| Experience    | 70%        | Hands-on practice and operational repetition |
| Communication | 20%        | Technical explanation and English exposure   |

The primary rule for June is:

> "Every theoretical concept must become a practical lab."

# June Operational Rules

## Focus Rules

* Avoid random tool exploration
* Avoid excessive course consumption
* Prioritize practical repetition
* Prioritize troubleshooting
* Prioritize consistency over intensity

---

## GitHub Rules

Every important lab must include:

* README documentation
* architecture explanation
* commands used
* troubleshooting notes
* lessons learned

---

## English Rules

English exposure must happen daily through:

* technical reading
* documentation writing
* architecture explanation
* listening practice
* technical vocabulary repetition

Perfect grammar is not the objective.

Technical clarity is the objective.
