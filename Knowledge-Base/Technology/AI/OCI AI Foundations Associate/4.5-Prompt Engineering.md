# 4.5-**Prompt Engineering**

First, let us define the terms. Prompt is the input or initial text provided to the large language model. Prompt engineering is the process of iteratively refining a prompt for the purpose of eliciting a particular style of response. Text prompts are how users typically interact with large language models.

Until now, what we have seen is the LLM models attempt to produce the next token or the next series of words that are most likely to follow from the previous set of words or the previous text. So, for example, if we give a prompt of something like four score and seven years ago-- this ago our, and this is the famous Gettysburg speech given by Abraham Lincoln in 1863-- what the LLM would do would be to complete this particular prompt.

So it's geared towards text completion because that is what we have seen in the previous-- the first lesson in introduction to large language models, that the language model is predicting the next set of words and completing the sentence. So it will complete this particular speech because it has seen that speech earlier.

Now that causes several challenges, because completion LLMs are trained to predict the next word on a large data set of internet text, rather than to safely perform the language task that the user wants. In this way, you cannot give instructions or ask questions to a completion LLM. Instead, you need to formulate your input as a prompt whose natural continuation is your desired output. And this is challenging. It's not possible all the time.

So what we do is something called instruction tuning. It's a critical step in LLM alignment, basically, which means that we want the LLM to give the kind of response we want. This process is quite advanced, and it involves fine-tuning a pre-trained LLM on a varied set of instructions, each paired with a desired output.

So, for example, there is this paper which was released by Meta on Llama 2. And in this-- if you go through the paper, basically what it states is Llama 2 foundational models were trained on a data set with something like two trillion tokens. And Llama 2 chat, which is specifically instruction tuned for chat, was additionally fine-tuned on something like 28,000 prompt response pairs created for this particular project. And there is a term which gets talked about called Reinforcement Learning from Human Feedback-- RLHF-- which is used to fine-tune LLMs to follow a broad class of written instructions.

In a nutshell, what RLHF is, it's a model training procedure that is applied to a fine-tuned language model to further align model behavior with human preferences and instruction following. So in this case, what we do is human annotators write prompts and compare the model outputs. The human feedback is subsequently used to train a reward model, which learns patterns in the preferences of the human annotators and can then automate preference decisions.

So that's basically how at a very high level, how RLHF works. So remember that most of the LLMs you interact with nowadays are instruction-tuned so it can perform the language task which the user wants rather than just complete the text prompts as it would do because these LLMs are nothing but completion LLMs.

Now, prompt engineering is challenging, as you just saw in the previous slide, but it can also be successful. There have been many successful strategies for generating prompts that are successful for specific tasks and particular models. Now we are going to look at few of them in the next two slides. The first one is in-context learning and also few-shot prompting.

Now, in-context learning, basically means that we are conditioning an LLM with instructions or demonstrations of the task it is meant to complete. Now, it is not learning in the traditional sense as none of the parameters of the model are changing. What in-context learning means is that it has demonstrations of the tasks that the model is supposed to complete. So-- and before we do that, there is also k-shot prompting, which is explicitly providing k examples of the intended task in the prompt.

So it could be 0-shot prompting, meaning you are providing zero examples, one-shot prompting-- one example, two-shot prompting. Two examples it could be k-shot meaning k examples. So what you see here in this example taken from the paper referenced below, that there are three examples followed by an example which you would like the model to complete. So it gives a task description, translate this text from English to French, and there are three examples.

So this is an example of a three-shot prompting. And then there is a prompt here, which is labeled as a prompt, and you want the model to translate cheese into French. So this is an example of in-context learning with three-shot prompting. And now based on the research which has gone in this area, few-shot prompting is widely believed to improve results over 0-shot prompting. So that's kind of quite well established nowadays.

The second successful prompting technique is what is referred to as chain-of-thought prompting. And if you have a complicated task, then what we are going to do is we are going to prompt the model to break the problem into small chunks, exactly like you would solve it if you were solving it yourself, and then you break it into smaller parts, and then you solve each of these intermediate steps or each of these smaller parts.

So this is basically what chain-of-thought prompting does. So it's basically including, in a nutshell, a reasoning step, describing the calculation logic to get to the final answer, before giving the final answer. So, for example, if you consider this particular prompt, it includes an answer. So it says something like Roger starts with five tennis balls. He buys two more cans of tennis balls. Each can has three tennis balls. So how many total tennis balls does he have?

And if you see this particular response here, it's actually giving a reasoning. And a verbal equation starts with 5, 2 cans of 3 is 6, and then 5 plus 6 is 11, and the answer is 11. So it's giving this particular answer. It's kind of like this chain of thought reasoning it is providing. And now we can ask the model another question.

Oracle cafeteria in building 250 had 20 bananas. They use 10 to make pancakes, and they purchased 8 more. How many bananas do they have now? And then you-- because you have given this chain-of-thought prompting, you would expect the model to give the right answer along with the reasoning. And that's what the model does. It says started with 20 minus 10 bananas. So this is the verbal equation. So now they have 10. And they bought 8 more, which means they have 10 plus 8, 18 bananas. Therefore, the final answer is 18.

Now if you don't give this chain-of-thought prompting, sometimes the model might return incorrect response. So both in-context learning, k-shot prompting, as well as chain-of-thought prompting, are two successful techniques which are used as part of prompt engineering. There are several other techniques, but this is a foundational course, so we're not getting into a bunch of those strategies.

Finally, one topic which is really important to know is around hallucination. Now, hallucination is basically means the model generated text that is non-factual and ungrounded. It's been defined in a number of different ways. For the purpose of our discussion, this is how we should define hallucination. It's text which is not grounded in any data that the model has been exposed to.

At times, statements that are nonsensical or factually incorrect are also considered to be hallucination. For example, consider the text on this slide, which contains a bold statement that is not factually correct. Specifically, it states that in the United States, people gradually adopted the practice to drive on the left side of the road. This is factually incorrect and would typically be considered a hallucination.

One thing to note about this example is the generated text is completely fluent and even starts off correct. Many people might believe that the model hallucinations are obvious and easy to identify, like the one on this slide. But a serious issue with hallucination is that sometimes hallucinations are very subtle and difficult to figure out.

Now there are some methods that are claimed to reduce hallucination. So, for example, retrieval augmented systems hallucinate less than 0-shot LLMs. And today there is no known methodology to reliably keep LLMs from hallucinating, but researchers are developing methods for measuring the groundedness of LLM generated output. So this is an area which has active research going on.

And that's pretty much it. In this lesson, we looked at prompt and prompt engineering. We looked at why it's challenging. We looked at what instruction-tuning is for LLMs. We looked at basically two successful strategies for prompt engineering, in-context learning, and few-shot prompting, and chain-of-thought prompting. And then finally, we looked at one issue with-- one challenge with prompt engineering around hallucination, and what can be done to reduce hallucination. I hope you found this lesson useful. Thanks for your time.

# 4.5- **Ingeniería de prompt**

Primero, definamos los términos. Un prompt es el texto inicial que se proporciona al modelo de lenguaje. La ingeniería de prompts es el proceso de refinar iterativamente un prompt con el fin de obtener un estilo de respuesta específico. Los prompts de texto son la forma en que los usuarios suelen interactuar con los modelos de lenguaje.

Hasta ahora, lo que hemos visto es que los modelos LLM intentan producir el siguiente token o la siguiente serie de palabras que tienen más probabilidades de seguir al conjunto de palabras anterior o al texto anterior. Así, por ejemplo, si damos una indicación como "hace ochenta y siete años, hace nuestro, y este es el famoso discurso de Gettysburg pronunciado por Abraham Lincoln en 1863", lo que haría el LLM sería completar esta indicación en particular.

Está orientado a la autocompletación de textos, como vimos en la lección anterior (la primera de la introducción a los modelos de lenguaje complejos): el modelo predice el siguiente conjunto de palabras y completa la oración. Por lo tanto, completará este discurso en particular porque ya lo ha visto antes.

Esto plantea varios desafíos, ya que los modelos de lenguaje de autocompletado se entrenan para predecir la siguiente palabra en un gran conjunto de datos de texto de internet, en lugar de realizar de forma segura la tarea lingüística que el usuario desea. Por lo tanto, no se pueden dar instrucciones ni hacer preguntas a un modelo de lenguaje de autocompletado. En cambio, es necesario formular la entrada como una indicación cuya continuación natural sea la salida deseada. Y esto es complejo. No siempre es posible.

Lo que hacemos es lo que se denomina ajuste de instrucciones. Es un paso fundamental en la alineación de modelos de lógica descriptiva (LLM), lo que significa que buscamos que el LLM proporcione el tipo de respuesta que deseamos. Este proceso es bastante avanzado e implica el ajuste fino de un LLM preentrenado con un conjunto variado de instrucciones, cada una asociada a una salida deseada.

Por ejemplo, existe un documento publicado por Meta sobre Llama 2. En él, se indica que los modelos fundamentales de Llama 2 se entrenaron con un conjunto de datos de aproximadamente dos billones de tokens. Además, el chat de Llama 2, específicamente optimizado para instrucciones de chat, se ajustó con unas 28 000 parejas de preguntas y respuestas creadas para este proyecto. Se menciona un término llamado Aprendizaje por Refuerzo a partir de la Retroalimentación Humana (RLHF, por sus siglas en inglés), que se utiliza para ajustar los modelos de lenguaje natural (LLM) para que sigan una amplia gama de instrucciones escritas.

En resumen, RLHF es un procedimiento de entrenamiento de modelos que se aplica a un modelo de lenguaje optimizado para alinear aún más su comportamiento con las preferencias humanas y la capacidad de seguir instrucciones. En este caso, los anotadores humanos escriben indicaciones y comparan los resultados del modelo. La retroalimentación humana se utiliza posteriormente para entrenar un modelo de recompensa, que aprende patrones en las preferencias de los anotadores humanos y puede automatizar la toma de decisiones sobre dichas preferencias.

Así es, básicamente, como funciona RLHF a grandes rasgos. Recuerda que la mayoría de los sistemas de aprendizaje de lenguaje con los que interactúas hoy en día están optimizados para realizar la tarea lingüística que el usuario desea, en lugar de simplemente completar las indicaciones de texto, como lo harían normalmente, ya que estos sistemas no son más que sistemas de autocompletado.

Como se vio en la diapositiva anterior, la ingeniería de indicaciones es compleja, pero también puede ser exitosa. Existen muchas estrategias eficaces para generar indicaciones que funcionan bien para tareas y modelos específicos. En las próximas dos diapositivas, analizaremos algunas de ellas. La primera es el aprendizaje en contexto y la generación de indicaciones con pocos ejemplos.

El aprendizaje en contexto consiste básicamente en condicionar un modelo de lenguaje natural (MLN) con instrucciones o demostraciones de la tarea que debe realizar. No se trata de aprendizaje en el sentido tradicional, ya que ninguno de los parámetros del modelo cambia. El aprendizaje en contexto implica que se proporcionan demostraciones de las tareas que el modelo debe completar. Antes de continuar, también está el método de k ejemplos, que consiste en proporcionar explícitamente k ejemplos de la tarea prevista en el enunciado.

Podría tratarse de una indicación de 0 ejemplos, es decir, no se proporcionan ejemplos; una indicación de un ejemplo, un ejemplo; o una indicación de dos ejemplos, dos ejemplos. Podría ser una indicación de k ejemplos. En este ejemplo, tomado del documento citado a continuación, se muestran tres ejemplos seguidos de un ejemplo que se desea que el modelo complete. Se proporciona una descripción de la tarea, se traduce este texto del inglés al francés y se incluyen tres ejemplos.

Este es un ejemplo de una secuencia de tres indicaciones. Aquí hay una indicación, etiquetada como tal, y queremos que el modelo traduzca "queso" al francés. Este es un ejemplo de aprendizaje en contexto con secuencia de tres indicaciones. Según las investigaciones realizadas en este campo, se cree que la secuencia de pocas indicaciones mejora los resultados en comparación con la secuencia de cero indicaciones. Esto está bastante consolidado hoy en día.

La segunda técnica de inducción exitosa es la que se conoce como inducción en cadena de pensamiento. Si tienes una tarea complicada, lo que haremos es indicarle al modelo que divida el problema en partes pequeñas, tal como lo harías tú mismo, y luego lo dividirás en partes más pequeñas, y luego resolverás cada uno de estos pasos intermedios o cada una de estas partes más pequeñas.

Básicamente, esto es lo que hace la incitación a la cadena de pensamiento. En resumen, incluye un paso de razonamiento, describiendo la lógica del cálculo para llegar a la respuesta final, antes de dar la respuesta final. Por ejemplo, si consideramos esta incitación en particular, incluye una respuesta. Dice algo así como: Roger comienza con cinco pelotas de tenis. Compra dos latas más de pelotas de tenis. Cada lata tiene tres pelotas de tenis. Entonces, ¿cuántas pelotas de tenis tiene en total?

Y si observan esta respuesta en particular, en realidad está dando un razonamiento . Una ecuación verbal comienza con 5, 2 latas de 3 son 6, y luego 5 más 6 es 11, y la respuesta es 11. Así que está dando esta respuesta en particular. Es como esta cadena de razonamiento que está proporcionando. Y ahora podemos hacerle otra pregunta al modelo.

La cafetería Oracle en el edificio 250 tenía 20 plátanos. Usaron 10 para hacer panqueques y compraron 8 más. ¿Cuántos plátanos tienen ahora? Y luego, como usted proporcionó esta cadena de pensamiento, esperaría que el modelo diera la respuesta correcta junto con el razonamiento. Y eso es lo que hace el modelo. Dice que comenzaron con 20 menos 10 plátanos. Esta es la ecuación verbal. Entonces, ahora tienen 10. Y compraron 8 más, lo que significa que tienen 10 más 8, 18 plátanos. Por lo tanto, la respuesta final es 18.

Si no se proporciona esta guía de razonamiento, el modelo podría arrojar una respuesta incorrecta. Por lo tanto, tanto el aprendizaje en contexto como la guía de k-shots, así como la guía de razonamiento, son dos técnicas exitosas que se utilizan en la ingeniería de indicaciones. Existen otras técnicas, pero este es un curso básico, así que no profundizaremos en ellas.

Finalmente, un tema crucial a considerar es el de las alucinaciones. Básicamente, una alucinación se refiere a un texto generado por el modelo que carece de fundamento y no se basa en datos reales. Se ha definido de diversas maneras. Para este análisis, la definición que utilizaremos es la siguiente: texto que no se fundamenta en ningún dato al que el modelo haya estado expuesto.

En ocasiones, las afirmaciones sin sentido o incorrectas también se consideran alucinaciones. Por ejemplo, observe el texto de esta diapositiva, que contiene una afirmación en negrita que no es correcta. En concreto, afirma que en Estados Unidos la gente adoptó gradualmente la práctica de conducir por la izquierda. Esto es incorrecto y, por lo general, se consideraría una alucinación.

Un detalle a destacar de este ejemplo es que el texto generado es completamente fluido e incluso comienza correctamente. Mucha gente podría creer que las alucinaciones del modelo son obvias y fáciles de identificar, como la de esta diapositiva. Pero un problema grave con las alucinaciones es que a veces son muy sutiles y difíciles de detectar.

Actualmente existen algunos métodos que, según se afirma, reducen las alucinaciones. Por ejemplo, los sistemas de recuperación aumentada generan menos alucinaciones que los modelos de aprendizaje de aprendizaje (LLM) sin entrenamiento previo. Si bien hoy en día no se conoce ninguna metodología para evitar de forma fiable las alucinaciones en los LLM, los investigadores están desarrollando métodos para medir la verosimilitud de la información generada por estos modelos. Por lo tanto, se trata de un área en la que se está llevando a cabo una investigación activa.

Y eso es prácticamente todo. En esta lección, analizamos las indicaciones y su ingeniería. Vimos por qué es un desafío. Vimos qué es la optimización de instrucciones para los modelos de aprendizaje de lenguaje. Vimos dos estrategias exitosas para la ingeniería de indicaciones: el aprendizaje en contexto y las indicaciones con pocos ejemplos, y las indicaciones basadas en cadenas de pensamiento. Finalmente, analizamos un problema —un desafío— relacionado con las alucinaciones en la ingeniería de indicaciones y qué se puede hacer para reducirlas. Espero que esta lección les haya sido útil. Gracias por su tiempo.

# 4.5 - **Prompt Engenharia**

Primeiramente, vamos definir os termos. Prompt é a entrada ou o texto inicial fornecido ao modelo de linguagem de grande porte. Engenharia de prompts é o processo de refinar iterativamente um prompt com o objetivo de obter um estilo específico de resposta. Os prompts de texto são a forma como os usuários normalmente interagem com modelos de linguagem de grande porte.

Até agora, o que vimos foi que os modelos LLM tentam produzir o próximo token ou a próxima série de palavras que provavelmente decorrem do conjunto de palavras anterior ou do texto anterior. Então, por exemplo, se dermos um estímulo como "há oitenta e sete anos — isto é o nosso, e este é o famoso discurso de Gettysburg proferido por Abraham Lincoln em 1863" — o que o LLM faria seria completar esse estímulo específico.

Portanto, ele é voltado para a conclusão de texto porque foi isso que vimos na lição anterior — a primeira lição da introdução a grandes modelos de linguagem —, em que o modelo de linguagem prevê o próximo conjunto de palavras e completa a frase. Assim, ele completará essa fala específica porque já a viu anteriormente.

Isso acarreta diversos desafios, pois os modelos de linguagem de conclusão (LLMs) são treinados para prever a próxima palavra em um grande conjunto de dados de texto da internet, em vez de executar com segurança a tarefa linguística desejada pelo usuário. Dessa forma, não é possível dar instruções ou fazer perguntas a um LLM de conclusão. Em vez disso, é preciso formular a entrada como um estímulo cuja continuação natural seja a saída desejada. E isso é desafiador. Nem sempre é possível.

O que fazemos é algo chamado ajuste de instruções. É uma etapa crucial no alinhamento do LLM, basicamente, o que significa que queremos que o LLM forneça o tipo de resposta que desejamos. Esse processo é bastante complexo e envolve o ajuste fino de um LLM pré-treinado em um conjunto variado de instruções, cada uma associada a uma saída desejada.

Por exemplo, existe um artigo publicado pela Meta sobre o Llama 2. Nele, basicamente, o que se afirma é que os modelos fundamentais do Llama 2 foram treinados em um conjunto de dados com cerca de dois trilhões de tokens. E o chat do Llama 2, que é especificamente ajustado para instruções de bate-papo, foi adicionalmente refinado com cerca de 28.000 pares de perguntas e respostas criados para este projeto específico. Há um termo bastante discutido chamado Aprendizado por Reforço a partir de Feedback Humano (RLHF, na sigla em inglês), que é usado para refinar os LLMs (Modelos de Aprendizado por Reforço) para seguir uma ampla gama de instruções escritas.

Em resumo, o RLHF é um procedimento de treinamento de modelo aplicado a um modelo de linguagem já ajustado para alinhar ainda mais o comportamento do modelo às preferências humanas e ao seguimento de instruções. Nesse caso, anotadores humanos escrevem instruções e comparam as respostas do modelo. O feedback humano é então usado para treinar um modelo de recompensa, que aprende padrões nas preferências dos anotadores humanos e pode, assim, automatizar as decisões de preferência.

Então, basicamente, em linhas gerais, é assim que o RLHF funciona. Lembre-se de que a maioria dos LLMs com os quais você interage hoje em dia são ajustados para instruções, de modo que possam executar a tarefa linguística desejada pelo usuário, em vez de apenas completar as instruções de texto, como fariam normalmente, pois esses LLMs nada mais são do que LLMs de preenchimento.

Agora, a engenharia de prompts é desafiadora, como você viu no slide anterior, mas também pode ser bem-sucedida. Existem muitas estratégias eficazes para gerar prompts que funcionam para tarefas e modelos específicos. Vamos analisar algumas delas nos próximos dois slides. A primeira é a aprendizagem contextual e a segunda é a geração de prompts com poucos exemplos.

o aprendizado em contexto significa basicamente que estamos condicionando um modelo de aprendizagem linear (LLM) com instruções ou demonstrações da tarefa que ele deve realizar. Não se trata de aprendizado no sentido tradicional, pois nenhum dos parâmetros do modelo está sendo alterado. O aprendizado em contexto significa que o modelo recebe demonstrações das tarefas que deve executar. Antes disso, é importante mencionar o método de k-shot prompting, que consiste em fornecer explicitamente k exemplos da tarefa pretendida no prompt.

Portanto, pode ser um prompt de 0 exemplos, ou seja, você não fornece nenhum exemplo; um prompt de um exemplo, ou um prompt de dois exemplos, ou ainda um prompt de k exemplos. Neste exemplo retirado do artigo referenciado abaixo, vemos três exemplos seguidos de um exemplo que você gostaria que o modelo completasse. A tarefa consiste em traduzir um texto do inglês para o francês, e são apresentados três exemplos.

Este é um exemplo de um estímulo de três etapas. Aqui temos um estímulo, identificado como tal, e queremos que o modelo traduza "queijo" para francês. Este é um exemplo de aprendizado contextual com estímulos de três etapas. Com base nas pesquisas realizadas nesta área, acredita-se que o uso de poucos estímulos (poucos exemplos) melhora os resultados em comparação com o uso de nenhum estímulo. Isso já está bem estabelecido atualmente.

A segunda técnica de estímulo bem-sucedida é o que chamamos de estímulo por cadeia de pensamento. Se você tiver uma tarefa complexa, o que faremos é estimular o modelo a dividir o problema em pequenas partes, exatamente como você o resolveria se estivesse resolvendo sozinho. Em seguida, você divide o problema em partes menores e resolve cada uma dessas etapas intermediárias ou cada uma dessas partes menores.

Então, basicamente, é isso que o método de raciocínio lógico faz. Em resumo, ele inclui uma etapa de raciocínio, descrevendo a lógica de cálculo para chegar à resposta final, antes de fornecer a resposta final. Por exemplo, se considerarmos esta pergunta específica, ela inclui uma resposta. Ela diz algo como: "Roger começa com cinco bolas de tênis. Ele compra mais duas latas de bolas de tênis. Cada lata contém três bolas de tênis. Então, quantas bolas de tênis ele tem no total?"

E se você observar esta resposta específica aqui, verá que ela está, na verdade, apresentando um raciocínio . Uma equação verbal começa com 5, 2 latas de 3 são 6, e então 5 mais 6 é 11, e a resposta é 11. Portanto, ela está fornecendo essa resposta específica. É como se estivesse apresentando uma linha de raciocínio. E agora podemos fazer outra pergunta ao modelo.

A cafeteria Oracle, no prédio 250, tinha 20 bananas. Eles usaram 10 para fazer panquecas e compraram mais 8. Quantas bananas eles têm agora? E então, como você forneceu essa linha de raciocínio, você esperaria que o modelo desse a resposta correta junto com o raciocínio. E é isso que o modelo faz. Ele diz: "Começaram com 20 menos 10 bananas". Então, esta é a equação verbal. Agora eles têm 10. E compraram mais 8, o que significa que têm 10 mais 8, 18 bananas. Portanto, a resposta final é 18.

Agora, se você não fornecer esse estímulo de cadeia de raciocínio, às vezes o modelo pode retornar uma resposta incorreta. Portanto, tanto o aprendizado contextualizado quanto o estímulo baseado em k exemplos, assim como o estímulo de cadeia de raciocínio, são duas técnicas bem-sucedidas usadas como parte da engenharia de estímulos. Existem várias outras técnicas, mas este é um curso introdutório, então não vamos nos aprofundar em muitas dessas estratégias.

Por fim, um tópico realmente importante a ser compreendido diz respeito à alucinação. Alucinação significa, basicamente, texto gerado pelo modelo que não é factual e não tem fundamento. Ela foi definida de diversas maneiras. Para os propósitos desta discussão, definiremos alucinação da seguinte forma: é um texto que não se baseia em nenhum dado ao qual o modelo tenha sido exposto.

Às vezes, afirmações sem sentido ou factualmente incorretas também são consideradas alucinações. Por exemplo, considere o texto neste slide, que contém uma afirmação em negrito que não é factual. Especificamente, afirma que nos Estados Unidos, as pessoas gradualmente adotaram a prática de dirigir do lado esquerdo da estrada. Isso é factualmente incorreto e normalmente seria considerado uma alucinação.

Um aspecto importante deste exemplo é que o texto gerado é completamente fluente e até começa corretamente. Muitas pessoas podem acreditar que as alucinações modeladas são óbvias e fáceis de identificar, como a deste slide. Mas um problema sério com as alucinações é que, às vezes, elas são muito sutis e difíceis de decifrar.

Existem alguns métodos que supostamente reduzem as alucinações. Por exemplo, sistemas de recuperação aumentada geram menos alucinações do que modelos de aprendizagem por aprendizagem (LLMs) sem nenhum estímulo. Atualmente, não existe uma metodologia conhecida para impedir de forma confiável que os LLMs gerem alucinações, mas pesquisadores estão desenvolvendo métodos para medir a precisão das informações geradas por esses modelos. Portanto, essa é uma área com pesquisas ativas em andamento.
